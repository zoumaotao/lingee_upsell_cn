主数据源（SoT）原则（国内版）：灵基 LBC 是产品/订单/订阅/席位/点数/权益/合同的权威源；金蝶云平台 是支付执行层（支付结果/流水以其 notify 回调为准）；TID/企业认证权威源在金蝶云账号体系，LBC 只读引用；SN/开票/收款记账权威源在 KBC，LBC 异步推事件后由 KBC 处理。两边以 kingdee_tid、kingdee_sn、notify_id 做系统间映射。

1 · 商品域（差异）仅写变化部分

PRODUCT 与 v3 通用版完全一致。SKU 差异：currency 固定为 CNY，list_price 以分（人民币）存储。GIFT_POINTS_RULE 与通用版完全一致。
SKU · 可售变体（国内版差异字段） SoT 灵基
字段类型说明（差异部分）
list_price / currencyint / char(3)国内修改 目录价（分，人民币）；currency 固定为 CNY（¥99/席/月 → 存 9900，年付存 118800，含税）
GIFT_POINTS_RULE 与 v3 通用版完全一致。赠送规则不区分国内外，运营后台统一配置，改后下周期生效，写 AUDIT_LOG（event_type=gift_rule_updated）。

2 · 客户域（差异）ACCOUNT 扩展

ACCOUNT 新增 TID 认证状态字段和统一社会信用代码；移除 stripe_customer_id；tax_status 改为增值税口径。
ACCOUNT · 购买主体（国内差异字段） SoT 灵基
以下仅列国内与 v3 通用版有差异的字段。其余字段（legal_name / lifecycle_stage / primary_contact / owner / dedup_status 等）与 v3 完全一致。
字段类型说明（差异部分）
① 身份（国内增量）
unified_social_credit_codestring(18)国内新增 统一社会信用代码（开票必填，来自金蝶云账号企业认证）。唯一约束，同企业只有一条 ACCOUNT
tax_status（v3 原字段）enum国内替换 v3 通用值（domestic_sst/export_zero_rated）→ 国内改为 vat_general（增值税一般纳税人）/ vat_small（小规模纳税人）
③ 映射（国内增量）
stripe_customer_idstring国内版移除（无 Stripe）
kingdee_tidstring国内核心 金蝶云账号 TID（企业级唯一标识，LBC 只读引用）。开通租户前必须就位。prospect 阶段可空，试用时绑定，新购前必须有。注意：v3 通用版 ACCOUNT 中 kingdee_tid 是可选映射；国内版将其视为核心必要字段
tid_certification_statusenum国内新增 certified（已完成企业认证）/ uncertified（有 TID 但认证未完成）/ none（无 TID，通常为 prospect 早期）
ACCOUNT.dedup_status：国内版 dedup_status 的 matched 含义是"已对齐 TID"，匹配链为：① 已绑定 TID 直接命中 → ② 统一社会信用代码 → ③ 企业名称模糊 → 人工复核。

3 · 销售域（差异）FREE_TRIAL_REQUEST 扩展

LEAD / OPPORTUNITY / CONTACT 与 v3 完全一致（LEAD 的 offline_event/existing_customer/pql_signal 已在 v3 通用版中）。FREE_TRIAL_REQUEST 新增 TID 相关字段。
FREE_TRIAL_REQUEST · 试用申请（国内差异字段） SoT 灵基
v3 通用版已有 contact_name / phone / company_name / cloud_account_type。国内版在此基础上新增 TID 绑定字段。
字段类型必填说明（差异部分）
kingdee_cloud_tidstring国内新增 金蝶云账号 TID。老客户填资时自动绑定；新客户代注册企业后获得（未认证状态）。可空（待注册接口确认）
tid_certification_statusenum国内新增 TID 对应企业的认证状态：certified（已认证）/ uncertified（有 TID 但认证未完成）/ unknown
tid_linked_attimestamp国内新增 TID 绑定时间（老客户留资时 / 新客户新购完成企业认证时）
failure_reason（v3 原枚举扩展）enum国内新增枚举 在 v3 原有枚举基础上新增 tid_registration_failed（TID 注册/绑定失败，国内特有）
试用→新购升级行为：新购开通成功后，试用租户 type 改为 sandbox（正式沙箱，数据保留），同时新建独立正式生产租户。不是试用租户直接升级为生产。

字段类型说明
（全部字段）—国内版无此实体，海外版请参见 v3 通用版文档
ORDER_EVENT · 订单状态历史（国内差异说明）
结构与 v3 完全一致。from_status / to_status 含义扩展：国内版新增 pending_activation 状态，ORDER_EVENT 记录进入和离开该状态的流转。

6 · 财税合规域（差异）重大修改：INVOICE / COMPLIANCE_CHECK；新增 KBC_INVOICE_REQUEST / SIGNED_CONTRACT

EINVOICE_SUBMISSION 由 KBC_INVOICE_REQUEST 替代。INVOICE 改为国内增值税口径。新增 SIGNED_CONTRACT（单签合同）。COMPLIANCE_CHECK 简化为本地黑名单+反欺诈。CONSENT_LOG / AGREEMENT_VERSION 与 v3 通用版完全一致。
INVOICE · 发票记录（国内差异字段） SoT 灵基（存引用，正文在 KBC）
LBC 不存发票正文，只存 KBC 返回的发票号/链接/状态。开票条件：企业认证完成（有统一社会信用代码）+ 开通成功（服务已交付）。

vat_typeenum国内新增 general（增值税普通发票）/ special（增值税专用发票）。客户在管理后台选择，也可事后申请修改

COMPLIANCE_CHECK · 合规/风控明细（国内简化） SoT 灵基
字段类型说明（差异部分）
check_typeenum国内简化 v3 通用版的 sanctions/export_control/risk_score/tax_id 四种 → 国内简化为：blacklist（本地黑名单，代替 sanctions/export_control，由运营手工维护封禁账户）/ risk_score（基础反欺诈，LBC 本地规则，非 Stripe Radar）。无 OFAC/制裁名单/出口管制检查

7 · 集成域（差异）WEBHOOK_EVENT → NOTIFY_EVENT；NOTIFICATION 扩展
